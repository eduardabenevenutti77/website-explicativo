Conheça um pouco sobre o processo de DevOps

É uma combinação de práticas e ferramentas que automotizam o processo entre as equipes de desenvolvimento e operações. Sendo os principais objetivos:

  1. Aumento na eficiência
  2. Aumento na qualidade dos produtos
  3. Reduzir o tempo de entraga dos projetos
  4. Utilização de feedbacks contínuos

Além disso, podemos validar a presença dos tópicos a seguir dentro do conceito de DevOps

  • Automação para facilitar a entrega contínua.
  • Integração e entrega contínuas (CI/CD) para aprimorar o desenvolvimento.
  • Monitoramento e logging para rápida detecção de problemas.
  • Colaboração entre equipes de desenvolvimento e operações.
  • Escalabilidade para ajustar recursos conforme necessário.

Qual é o conceito de GitHub e GitHub Pages:

GitHub é uma plataforma que combina funcionalidades de controle de versão, hospedagem de repositórios e interação social entre desenvolvedores. Ele permite que equipes colaborem e compartilhem códigos usando o Git, um sistema de controle de versão distribuído.

GitHub Pages, por outro lado, é uma ferramenta específica do GitHub que permite a criação e hospedagem de páginas estáticas diretamente de um repositório.

Podemos pensar no GitHub como uma plataforma para organizar e gerenciar "containers de código". Assim como o Docker permite empacotar, compartilhar e gerenciar aplicativos em contêineres padronizados, o GitHub organiza, controla e distribui versões de arquivos de código e texto de maneira centralizada.

  1. GitHub (plataforma principal) é como o Docker Hub (encontra imagens de contêiner prontas para usar), que armazena e compartilha contêineres.
  2. GitHub Pages é como executar um contêiner específico. Exemplo: Pense em um contêiner Docker configurado apenas para servir um site estático, como uma imagem do NGINX que hospeda arquivos HTML. O GitHub Pages desempenha esse papel: pega arquivos estáticos do repositório e os "executa" para torná-los acessíveis ao público.

Como utilizar o GitHub Pages?

Passo 1: Criar um Repositório no GitHub

  1. Acesse o GitHub: Entre na sua conta do GitHub ou crie uma, se ainda não tiver.
  2. Novo Repositório: Clique no botão de “+” no canto superior direito e selecione “New repository”.
  3. Nome do Repositório: Dê um nome ao seu repositório. Para hospedar em username.github.io, o repositório deve ter o nome exatamente igual ao seu nome de usuário.
  4. Configurações do Repositório:
    • Escolha se o repositório será público ou privado (público é necessário para GHPages gratuito).
    • Marque a opção para inicializar com um arquivo README, se desejar.
  5. Criar Repositório: Clique em “Create repository”.

Passo 2: Adicionar Arquivos do Projeto

  1. Upload dos Arquivos: Na página do repositório, clique em “Add file” e depois em “Upload files”. Você pode arrastar e soltar seus arquivos HTML, CSS, JavaScript e outros.
  2. Commit dos Arquivos: Após adicionar os arquivos, role para baixo e clique em “Commit changes” para salvar as alterações.

Passo 3: Habilitar o GitHub Pages

  1. Configurações do Repositório: Vá para a aba “Settings” do seu repositório.
  2. GitHub Pages: Role até encontrar a seção “GitHub Pages”.
  3. Fonte: Na seção “Source”, selecione a branch main (ou master, dependendo da configuração) e a pasta onde estão os arquivos (geralmente root).
  4. Salvar: Clique em “Save”. Após alguns momentos, você verá um link para acessar seu site.

Passo 4: Acessar seu Site

Após habilitar o GitHub Pages, seu site estará disponível em https://username.github.io/nome-do-repositorio.

Gerenciamento de Processos Node.js com PM2, passo a passo para a utilização:

1. Instalar o PM2

Instale o PM2 globalmente usando o seguinte comando:

npm install -g pm2

2. Iniciar a Aplicação com PM2

Inicie sua aplicação Node.js com o comando abaixo:

pm2 start app.js --name "minha-app"

Explicação: O parâmetro --name define um nome para o processo.

3. Monitorar Processos

Use o seguinte comando para visualizar os processos gerenciados pelo PM2:

pm2 list

4. Reinicialização Automática

O PM2 irá reiniciar automaticamente o processo se ele cair. Para garantir que ele reinicie após o servidor reiniciar, execute o comando:

pm2 startup

Isso configurará o PM2 para iniciar automaticamente após um reboot.

5. Configurar Watch para Auto-Restart

Ative a reinicialização automática com o comando abaixo, que monitora alterações nos arquivos:

pm2 start app.js --watch --name "minha-app"

6. Monitoramento Avançado

Para monitoramento em tempo real dos processos, use o comando:

pm2 monit

7. Ver Logs em Tempo Real

Veja os logs da sua aplicação com o comando:

pm2 logs minha-app

8. Usar o Ecosystem File

Para múltiplas aplicações, você pode usar o arquivo de configuração ecosystem.config.js. Isso facilita o gerenciamento de várias instâncias do PM2 em um único arquivo. Veja um exemplo básico:

          module.exports = {
            apps : [{
              name: 'minha-app',
              script: 'app.js',
              watch: true,
              instances: 'max',
              exec_mode: 'cluster'
            }]
          };

Como criar uma imagem docker do APP criado com o PM2 e Node.js

1. Criar o Dockerfile

O primeiro passo é criar um arquivo chamado Dockerfile no diretório raiz do seu projeto com o seguinte conteúdo:

          FROM node:16
          WORKDIR /app
          COPY package*.json ./
          RUN npm install
          COPY . .
          EXPOSE 3000
          CMD ["npm", "start"]

Isso define como a imagem Docker será construída para sua aplicação Node.js.

2. Construir a Imagem Docker

Agora, você pode construir a imagem Docker com o seguinte comando:

docker build -t minha-app .

Esse comando cria a imagem a partir do Dockerfile que você criou.

3. Rodar o Contêiner

Para rodar a aplicação dentro de um contêiner, execute:

docker run -p 3000:3000 minha-app

Isso mapeia a porta 3000 do contêiner para a porta 3000 da máquina local.

Benefícios da Containerização com Docker:

  • Portabilidade -> A aplicação pode ser executada em qualquer lugar, desde que o Docker esteja instalado.
  • Isolamento -> Cada contêiner roda em seu próprio ambiente, evitando conflitos de dependências.
  • Escalabilidade -> É fácil escalar sua aplicação rodando múltiplos contêineres.
  • Facilidade de Implantação -> A implantação em diferentes ambientes se torna muito mais simples e confiável.
  • Desempenho -> Os contêineres são mais leves e rápidos do que máquinas virtuais.
  • Facilidade de Manutenção -> Com contêineres, atualizar e gerenciar sua aplicação é mais fácil.
  • Segurança -> O isolamento de contêineres aumenta a segurança, limitando a exposição de falhas.

Veja como utilizar o dockerfile e o docker compose em múltiplos contêiners

1. Dockerfile

O Dockerfile define como a aplicação será configurada dentro do contêiner. A seguir está o exemplo para uma aplicação Node.js.

          FROM node:16
          # Definir o diretório de trabalho dentro do contêiner
          WORKDIR /usr/src/app

          # Copiar os arquivos de package.json e package-lock.json para instalar as dependências
          COPY package*.json ./

          # Instalar as dependências da aplicação
          RUN npm install

          # Copiar o restante dos arquivos da aplicação
          COPY . .

          # Expor a porta que a aplicação vai rodar
          EXPOSE 3000

          # Comando para rodar a aplicação
          CMD ["npm", "start"]

2. docker-compose.yml

O docker-compose.yml define e configura múltiplos contêineres para orquestrar o ambiente de desenvolvimento ou produção.

            version: '3.8'

            services:
            app:
              build: .
              ports:
                - "3000:3000"
              environment:
                - MYSQL_HOST=db
                - MYSQL_USER=root
                - MYSQL_PASSWORD=my-secret-pw
                - MYSQL_DATABASE=mydb
              depends_on:
                - db

                  db:
                  image: mysql:8
                  environment:
                    MYSQL_ROOT_PASSWORD: my-secret-pw
                    MYSQL_DATABASE: mydb
                  ports:
                    - "3306:3306"

3. Como usar

  1. Crie o arquivo Dockerfile no diretório do seu projeto.
  2. Crie o arquivo docker-compose.yml no mesmo diretório.
  3. Construa os contêineres:
    4. docker-compose build
  4. Suba os contêineres:
    5. docker-compose up

    A aplicação Node.js estará disponível na URL http://localhost:3000 e poderá se conectar ao banco de dados MySQL.
