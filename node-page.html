Como Funciona o Node:

Node.js é uma plataforma de execução de JavaScript no lado do servidor. Ele permite que desenvolvedores utilizem JavaScript para construir aplicações backend, o que antes era feito apenas no navegador. Node.js é baseado no motor V8 do Google Chrome, que compila o JavaScript para código de máquina, tornando-o rápido e eficiente.

Uma das principais características do Node.js é sua arquitetura orientada a eventos e seu modelo de I/O (entrada/saída) não bloqueante, o que o torna ideal para aplicações que exigem alta escalabilidade e trabalham com muitas requisições simultâneas, como servidores de aplicativos em tempo real, APIs, e sistemas de streaming de dados.

Com um grande ecossistema de bibliotecas disponíveis através do npm (Node Package Manager), Node.js é amplamente utilizado para desenvolver aplicações modernas e escaláveis, incluindo sistemas web e serviços baseados em microservices. Ele fornece uma vasta biblioteca de módulos para criar aplicações, incluindo funcionalidades como servidores HTTP, manipulação de arquivos e integração com bancos de dados.

Requisitos para usar Node.js

Instalar o Node.js

  • Faça o download do instalador no site oficial: https://nodejs.org.
  • O instalador inclui o npm (Node Package Manager).

Ambiente de Desenvolvimento

  • Um editor de código, como Visual Studio Code, é recomendado.

Terminal de Linha de Comando

  • Acesse o terminal do sistema (Prompt de Comando, PowerShell, ou Terminal no Linux/macOS).

Conhecimento Básico de JavaScript

  • É essencial para programar no ambiente Node.js.

Configuração do Caminho

  • Após a instalação, certifique-se de que o comando node -v funciona no terminal, indicando que o Node.js está no PATH do sistema.

Opcional: Banco de Dados

  • Caso sua aplicação precise armazenar dados, um banco de dados como MySQL, MongoDB ou outros pode ser necessário.

Utilização de Módulos Principais:

Os módulos principais do Node.js, como http, fs, e path, permitem criar funcionalidades básicas.

O módulo http é usado para criar servidores web e lidar com requisições HTTP. Ele fornece uma interface para construir aplicativos baseados em protocolos HTTP, como APIs ou páginas web.

Os principais recursos são:

  • Criar servidores HTTP ou HTTPS.

  • Processar métodos HTTP como GET, POST, PUT, DELETE.

  • Manipular cabeçalhos e status de resposta.

    • Aqui esta um exemplo de como funciona:

    Para copiar, clique no ícone ao lado

    1 const http = require('http'); 2 3 const server = http.createServer((req, res) => { 4     if (req.method === 'GET') { 5         res.writeHead(200, { 'Content-Type': 'text/plain' }); 6         res.end('Resposta para uma requisição GET'); 7     } else { 8         res.writeHead(405); 9         res.end('Método não permitido'); 10     } 11 }); 12 13 server.listen(3000, () => { 14     console.log('Servidor rodando em http://localhost:3000'); 15 });

    No exemplo acima o código cria um servidor HTTP com módulo http do Node.js. Ele Processar requisições na porta 3000 e responde de forma diferente dependendo do método HTTP:

    • Para requisições GET, retorna uma mensagem de sucesso com status 200: "Resposta para uma requisição GET".

    • Para outros métodos, retorna o status 405: "Método não permitido".

    O servidor começa a rodar e exibe a mensagem: "Servidor rodando em http://localhost:3000".

    Exemplo de estrutura express.js:

    O Express.js é um framework para simplificar a criação de aplicações Node.js.

    Para copiar, clique no ícone ao lado

    1 const express = require('express'); 2 const app = express(); 3 4 app.use(express.json()); // Middleware para JSON 5 6 // Rota Principal 7 app.get('/', (req, res) => { 8     res.send('Bem-vindo ao servidor Express!'); 9 }); 10 11 // Rota de exemplo para POST 12 app.post('/dados', (req, res) => { 13     const dados = req.body; 14     res.json({ mensagem: 'Dados recebidos!', dados }); 15 }); 16 17 // Iniciando o servidor 18 app.listen(3000, () => { 19     console.log('Servidor rodando em http://localhost:3000'); 20 });

    O que ele faz?

    • Configura rotas para manipular diferentes requisições HTTP (GET, POST, etc.).

    • Usa middlewares para processar dados, como JSON.

    Integração com Banco de Dados

    A integração com bancos de dados pode ser feita usando bibliotecas como Mongoose (para MongoDB) ou Sequelize (para PostgreSQL e outros).

    Exemplo com Sequelize e PostgreSQL

    Para copiar, clique no ícone ao lado

    Práticas de Segurança

    Middleware para Autenticação e Autorização

    Autenticação é o processo de verificar a identidade do usuário. Autorização define quais recursos ele pode acessar.

    Para copiar, clique no ícone ao lado

    1const express = require('express'); 2const jwt = require('jsonwebtoken'); // Biblioteca para autenticação com tokens 3 4const app = express(); 5const SECRET = 'chave-secreta'; 6 7// Middleware de autenticação 8function autenticarToken(req, res, next) { 9     const token = req.headers['authorization']; 10      if (!token) return res.sendStatus(401); 11 12      jwt.verify(token, SECRET, (err, usuario) => { 13        if (err) return res.sendStatus(403); 14        req.usuario = usuario; 15        next(); 16     }); 17} 18 19// Rota protegida 20app.get('/protegido', autenticarToken, (req, res) => { 21     res.send(`Olá, ${req.usuario.nome}, você acessou uma rota protegida!`); 21}); 22 app.listen(3000, () => { 23     console.log('Servidor rodando em http://localhost:3000'); 24});

    O que ele faz?

    • Protege rotas com JWT (JSON Web Token).

    • Usa um middleware para verificar a presença e validade do token.

    O Node.js é uma plataforma para desenvolvimento backend que permite criar servidores rápidos e escaláveis utilizando JavaScript.

    Seu principal diferencial é o modelo não bloqueante e assíncrono, o que significa que ele pode processar múltiplas requisições simultaneamente sem esperar que uma operação termine para começar outra.

    Isso faz com que o Node.js seja extremamente em aplicações que exigem alta performance, como sistemas em tempo real e APIs com grande volume de usuários. Além disso, com seu motor V8 e arquitetura single-threaded, o Node.js oferece alta performance ao mesmo tempo em que utiliza poucos recursos, tornando-o uma excelente opção para criar servidores ágeis e escaláveis.
